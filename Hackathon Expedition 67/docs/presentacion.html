El Problema

¿Por qué existe una brecha?

Empresas Donantes

Tienen presupuesto de Responsabilidad Social Empresarial, pero no saben a quién donar de manera segura y sin alta burocracia.

Zonas Vulnerables

Hospitales rurales y escuelas en el país tienen necesidades urgentes (agua, techo, útiles), pero carecen de visibilidad nacional.

La Solución

Plataforma Tecnológica

Una App web Inteligente con 3 pilares clave:

Tablones Directos

Las entidades publican requerimientos exactos sin intermediarios.

Mapa Geográfico

Visualización interactiva de urgencias y zonas sin cobertura.

Verificación

Filtro anti-fraude para asegurar donaciones 100% reales.

Incentivos de Donación

Gamificamos la solidaridad corporativa en Colombia.

  • Ranking Público: Visibilidad y reconocimiento de marca a nivel nacional e internacional.
  • Categorías Oficiales: Medallas dinámicas y certificados virtuales (Oro, Plata, Bronce).
  • Incentivo Fiscal: Reducción auditada del impuesto de renta avalada por Integración con Min. Hacienda y DIAN.

Funcionamiento

¿Cómo funciona?

Cerrando la brecha en 3 simples pasos

1

Registro Geográfico

La entidad o zona rural se registra y ubica en el mapa interactivo (Leaflet.js).

2

Tablones Directos

Se publican necesidades específicas y su nivel de urgencia, sin intermediarios.

3

Matching & Donación

La empresa visualiza, dona y el sistema automatiza el reconocimiento y beneficio fiscal.

Arquitectura

Engine Técnico Base

Web Frontend SPA responsiva, Vanilla JS Modular, CSS3 nativo avanzado para Ui/Ux, y visores Leaflet.js para los mapas.
Backend API Node.js usando Arquitectura Limpia para el manejo seguro de tablones, subidas y las transacciones.
Databases PostgreSQL (DB Relacional Central Robusta) + Redis (Caché rápida de uso de datos variables como Rankings).
Ext. Integrations Pasarelas de Pago Cifradas y Registro DIAN.
Atributos de Calidad

Propuesta de Valor Técnica

Diseño estructurado bajo estándares de Ingeniería (SAD)

Usabilidad & a11y

UI fluida, inclusiva y lista para usuarios rurales. Alta accesibilidad de lectura (W3C).

Privacidad

Cumplimiento de la Ley 1581 (Col). Anonimización de datos de entidades vulnerables.

Rendimiento

Vanilla JS sin frameworks pesados, garantizando acceso en redes inestables (3G).

Escalabilidad

Arquitectura modular orientada a servicios para fácil migración a un backend REST.
